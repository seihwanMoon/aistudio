## ⚡ 코딩 에이전트 사용 지침

### 작업 시작 전 반드시 확인
1. `PROGRESS.md` 열어서 현재 완료된 항목 확인
2. 해당 Phase 파일 (`PHASE1_MVP.md` 등) 에서 세부 스펙 확인
3. `ARCHITECTURE.md` 에서 파일 경로 및 구조 확인

### 작업 완료 시 반드시 실행
1. `PROGRESS.md` 에서 해당 항목 `[ ]` → `[x]` 체크
2. 작업 날짜 기입
3. 블로커/이슈 있으면 해당 항목 아래에 `> ⚠️ 이슈:` 로 기록

### 파일 생성 규칙
- 모든 파일은 `ARCHITECTURE.md` 의 폴더 구조를 따를 것
- 절대 폴더 구조를 임의로 변경하지 말 것
- 한국어 UI 텍스트는 별도 `frontend/src/constants/korean.js` 에 모아서 관리

## 🔑 핵심 기술 결정사항 (변경 금지)

| 결정 | 이유 |
|------|------|
| 브라우저 기반 로컬 웹앱 (Electron 아님) | 배포/업데이트 단순화 |
| SQLite (PostgreSQL 아님) | 설치 불필요, 파일 기반 |
| FLAML AutoML (AutoGluon 아님) | CPU 환경에서 가장 빠름 |
| SSE for 학습 진행률 (WebSocket 아님) | 단방향 스트림에 SSE가 적합 |
| WeasyPrint for PDF | 한국어 지원 오픈소스 |
